1、单页应用程序：SPA-Single Page Application

定义：所有功能在一个html页面实现
单页：网易云音乐，在一个页面跳转
页面性能高，开发效率高，用户体验好（最大原因：按需更新），学习成本高，首屏加载慢，SEO差
多页：京东、淘宝，在多个页面切换

2、路由入门

1.定义

路由是一直映射关系
vue中路由：路径和组件的映射关系
通过路由知道不同路径应该匹配渲染哪个组件

2.VueRouter

1.说明：Vue官方的一个路由插件，是一个第三方包

2.作用：修改地址栏路径时，切换显示匹配的组件

3.使用（5+2）

1.5个基础步骤

1.下载vuerouter模块

Vue2使用VueRouter3.x Vuex3.x
Vue3使用VueRouter4.x Vuex4.x
npm install vue-router@3.6.5

3.安装注册（vue插件要先安装注册）

4.创建路由对象

5.注入，将路由对象注入到new Vue实例中，建立关联

2.2个核心步骤

1.创建需要的组件（views目录），配置路由规则

path是路由规则里的路径匹配标识，它决定了浏览器地址栏里的路径、和那个组件对应

2.配置导航，配置路由出口（路由匹配的组件显示位置）

3.组件存放目录问题（组件分类）

.vue文件分为2类：页面组件&复用组件
src/views文件夹
页面组件-页面展示-配合路由用
src/components文件夹
复用组件-展示数据-常用于复用

1.导航链接

需求：实现导航高亮效果
vue-router提供了一个全局组件router-link（取代a标签）
1.能跳转，配置to属性指定路径（必须）。本质还是a标签，to无需#
2.能高亮，默认就会提供高亮类名，可以直接设置高亮样式，可以通过控制台看类名

2.两个类名

1.router-link-active模糊匹配（用的多）
to=‘/my ’可以匹配/my /my/a /my/b ….

2.router-link-exact-active精确匹配
to=‘/my ’只可以匹配/my

2.Vue路由

1.Vue路由-重定向

问题：网页打开，url默认是/路径，未匹配到组件时，会出现空白
说明：重定向-》匹配path后，强制跳转path路径
语法：{path:匹配路径，redirect：重定向到的路径}

2.Vue路由-404

作用：当路径找不到匹配时，给个提示页面
位置：配置在路由最后
语法：{path：”*”(任意路径-前面不匹配就命中最后这个)，component:对应组件}

3.Vue路由-模式设置

问题：路由路径看起来不自然，有#，能否切成真正路径形式
hash路由（默认），例如：http://localhost:8080/#/home
history路由（常用），例如：http://localhost:8080/home

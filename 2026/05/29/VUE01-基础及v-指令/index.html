3、Diff算法及VNode

Diff算法

Diff算法（Vue2核心，作用：对比新旧虚拟DOM，最小化更新真实DOM）

  1. 作用
    数据变了 → 生成新VNode（新虚拟DOM），不和旧DOM全量替换；
    Diff 对比新旧虚拟DOM树，只改不一样的DOM，减少DOM操作开销。
    没有Diff：每次更新全删整个页面DOM、重新创建，性能极差。

  2. Vue2 Diff规则：同层比较（不跨层级比对）
    只在同一层级节点挨个对比，不会跨父节点找节点，降低复杂度。

  3. 三步对比：
    1.节点类型不一样：直接销毁旧DOM，新建DOM

    • 旧 <div>  → 新 <p> ：整段替换

    2.标签相同、属性不同：只更新属性，DOM节点复用

    •  <div class="a"> → <div class="b"> ：只改class

    3.标签一样、key存在：用key精准匹配列表节点（v-for必备）

    • 列表增删时，复用原有DOM，只移动位置，不重复创建销毁

  4. key的意义
    key 是节点唯一标识，Diff靠key快速找到新旧对应的节点；
    不加key默认用下标index，数组乱序、插入开头会引发DOM错误复用。

  5. 时机（什么时候走Diff）
    1. 初始化挂载：不走Diff，直接创建DOM
    2. 响应式数据修改（state/data变了）→触发视图更新→生成新VNode→执行Diff→局部更新真实DOM

  6. 总结
    Diff = 找差别 → 精准局部改DOM，少删少建DOM，优化渲染性能

VNode（虚拟节点 / Virtual Node，虚拟DOM）

1、定义
用JS对象描述真实DOM节点，这个JS对象就是VNode，不是页面上真实DOM元素。
真实DOM：
<div id="box">hello</div>

对应VNode（JS对象）

2、VNode从哪来？
render(h) → h() 就是创建VNode 

Vue挂载时：VNode → 翻译成真实DOM插入页面。

3、和Diff关系
数据更新 → 生成新VNode，拿新VNode和旧VNode Diff对比，算出差异，只局部改真实DOM。

4、关键区分

  • VNode：JS对象（内存里） → 虚拟DOM
  • el：真实DOM（页面上HTML标签）

5、挂载流程串联

3、挂载

Vue2 $mount(‘#app’) 完整流程

  1. 编译阶段
    template模板编译为render函数，h函数生成VNode。
    render函数是专门用来生成VNode的函数，在挂载阶段（mount）执行一次，数据发送变化时再次执行，生成新的VNode
  2. new Vue() 创建根实例
    初始化：data、computed、watch、生命周期；传入store，挂载 this.$store 到实例原型，全局子组件可访问。
  3. beforeMount 触发
    模板编译完成，真实DOM未创建、未挂载页面。
  4. 调用render生成VNode
    执行render(h => h(App))，生成整棵虚拟DOM树。
  5. patch创建真实DOM
    首次挂载无旧VNode，直接依据VNode生成原生DOM。
  6. 替换#app挂载容器
    用新DOM覆盖页面<div id="app">内部内容。
  7. mounted 触发
    DOM已渲染至页面，可操作DOM、发起网络请求。

总结
模板→render→VNode→patch生成DOM→替换挂载容器→mounted。

补充
- 初始化挂载只执行一次；
- 后续数据更新：新VNode → Diff对比 → patch局部更新DOM，不再走完整挂载。

Vue3 挂载 .mount() 流程

  1. 编译阶段（构建时/运行时）
    把template模板 → 编译成 render 渲染函数，生成 VNode（虚拟DOM）。

  2. 创建组件实例
    createApp(App) 生成应用实例、根组件实例，初始化：
    - props、setup 执行、ref/reactive 生成响应式数据
    - 注册生命周期、指令、组件

  3. beforeMount 钩子触发
    DOM还没生成、没插入页面，拿不到真实DOM。

  4. 虚拟DOM → 真实DOM（patch）
    执行 render() 得到VNode，通过 patch 算法：
    虚拟DOM转为真实DOM元素。

  5. 替换挂载点容器
    .mount(‘#app’) ：
    用生成的真实DOM替换 <div id="app">内部内容。

  6. onMounted 钩子触发
    DOM已挂载到页面，可以获取DOM、发接口请求。

  7. 总结
    模板→render→VNode→真实DOM→替换挂载节点→mounted。
    补充小知识点
    - 挂载只执行一次；
    - 更新不走挂载，只走 patch diff 更新DOM。

4、vue指令

带有v-前缀的特殊标签属性

1.v-show

作用：控制元素显示隐藏
本质是控制display：none
场景：频繁切换隐藏的场景

2.v-if

作用：控制元素显示隐藏
本质是基于添加判断创建和移除元素节点
场景：要么显示，要么隐藏

3.v-else/v-else-if

作用：辅助v-if进行判断渲染
语法：v-else v-else-if=”表达式”
注意：需要紧挨v-if一起使用

4.v-on

作用：注册事件 = 添加监听 + 提供处理逻辑

语法：
1、v-on：事件名=‘内联语句’
2、v-on：事件名=‘methods中的函数名’

可以简写写成@事件名

methods里写函数
this指向vue实例
调用传参：
可以通过调用进行传递参数

5.v-bind

作用：动态的设置html标签属性
语法：v-bind：属性名=‘表达式’
可以简写
：属性名=‘表达式’
因为不能通过插值表达式动态修改属性值，所以可以使用v-bind

6.v-for

作用：基于数据循环，多次渲染整个元素，包括标签
语法：
v-for =“（item，index）in 数组”

7.key

作用：给元素加唯一标识，便于vue进行列表项的正确排序复用
不加key：v-for的默认行为会尝试原地修改元素（就地复用）
注意点：
1.key的值只能是字符串或数字类型
2.key的值必须具有唯一性
3.推荐id作为key（唯一），不推荐index（会变化）

8.v-model

作用：给表单元素使用，双向数据绑定->可以快速获取或设置表单元素内容
数据变化：视图自动更新
视图变化：数据自动更新
语法：
v-model = “变量名”

v-html

作用：将字符串解析出html代码渲染，而不是作为普通文本显示

Vue04-生命周期及工程化
2026-06-10 14:56:23 # Vue #生命周期 #工程化

1、生命周期

什么时候发送初始化渲染请求（越早越好），什么时候开始操作dom

1.vue生命周期：

一个vue实例从创建到销毁的整个过程
生命周期四个阶段：
1、创建
数据变为响应式数据
2、挂载
渲染模板
3、更新
修改数据，更新视图
4、销毁
销毁实例

2.生命周期钩子

生命周期过程中，会自动运行一些函数，被称为生命周期钩子

作用：人开发者可以在特定阶段运行自己的代码

一共有八个，在各个阶段前后
beforeCreate、created（发送初始化渲染请求）、beforeMount、mounted（操作dom）、beforeUpdate（数据修改了，视图还未更新）、updated（数据修改了，视图已经更新）、beforeDestroy（释放vue以外的资源）、destroyed

3.饼图更新

echarts.setOption({…})
只更新要修改的数据

2、工程化开发入门

工程化开发模式：基于构建工具（例如：webpack）的环境开发Vue

1.Vue CLI

vue官方提供的全局命令工具
可以帮助我们快速创建一个开发vue项目的标准化基础架子（集成了webpack配置）

好处：
1.开箱即用，零配置
2.内置babel等工具
3.标准化

使用步骤：
1.全局安装（一次）（管理员权限下）：yarn global add @vue/cli 或 npm i @vue/cli -g
2.查看vue版本：vue –version
3.创建项目架子：vue create project-name（项目名-不能用中文，要在管理员权限下创建）
4.启动项目：yarn serve 或 npm run serve（找package.json）

脚手架目录

2.组件化开发&根组件

1.组件化
一个页面可以拆分成一个个组件，每个组件可以有自己独立的结构、样式、行为
好处：便于维护、利于复用->提升开发效率
组件分类：普通组件、根组件

2.根组件
整个应用最上层的组件，包裹所有普通小组件
app根组件-头部组件、导航组件、主体组件（商品组件、商品组件）、底部组件

3.app.vue文件的三个组成部分

template：结构（有且只有一个根元素）
script：js逻辑
style：样式（可支持less，需要装包）
给style标签添加属性lang=‘less’开启less功能
装包：yarn add less less-loader

4.普通组件注册

注册方式

1.局部注册：
只能在注册的组件内使用

1.创建.vue文件（三个组成部分）(写在conponents文件里)

2.在使用的组件内导入并注册

使用：使用的组件内导入
局部注册：

当html标签使用<组件名></组件名>
比如：
组件名规范：大驼峰命名

2.全局注册
所有组件内可以使用
1.创建.vue文件
2.main.js中进行全局注册
使用：在main.js里导入并进行全局注册Vue.component（’组件名’，组件对象）//一次注册一个，要注册多个可以写多行

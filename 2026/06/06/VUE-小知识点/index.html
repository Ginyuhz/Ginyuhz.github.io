符号$含义

  1. 定义
    $是Vue内部约定的标识
    凡是Vue自动挂载到组件this上的内置全局API/实例属性，名字前面统一加$,用来区分自定义变量
  1. 为什么要用  $ ？
    避免命名冲突：
    自己可以在data定义 store 变量： this.store=”xxx” ，
    Vue的系统API用 $store ，不会和自定义变量重名。
  2. 延伸小知识点
    1.$ 只是命名规范，没有JS语法特殊作用，不是运算符；
    2.Vue原型挂载原理：
  1. 额外：其他场景$
  • jQuery  $(‘#id’) ：$是jQuery对象简写；
  • JS/ES6  ${变量} ：$是模板字符串语法，和Vue这个无关。

this.$store和$store

JS 环境中，要通过this拿到当前组件实例，再访问$store,所以使用this.$store访问
Vue 模板会自动把组件实例暴露在作用域里，语法规定省略this，直接使用$store访问

  1. 核心概念
     $store是Vue给所有组件实例全局挂载的属性，指向Vuex 仓库对象。
  •  this  = 当前 Vue 组件实例
  •  this.$store ：在组件 JS 代码里访问仓库（必须写this ）
  •  $store ：在 模板 HTML 里直接使用（不用写this ）
  1. 分场景实战对比
    1. 在  <script>  脚本里（JS 逻辑）
      必须写  this.$store 
      JS 环境中，要通过  this  拿到当前组件实例，再访问  $store 。
    1. 在  <template> 模板里（HTML 部分）
      直接写  $store ，不用this 
      Vue 模板会自动把组件实例暴露在作用域里，语法规定省略  this 。

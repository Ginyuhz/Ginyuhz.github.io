VUE18-模块module
2026-06-08 17:40:20 # VUE #模块module #访问模块

作用

模块module，就是把大型store拆分成多个独立的小模块，每个模块都有自己的state、mutations、actions、getters，解决单文件store臃肿、命名冲突的问题
好处：

  1. 代码拆分：拆分成模块后，每个模块只负责自己的业务
  2. 命名隔离：模块可以配置命名空间避免冲突
  3. 可维护性：业务逻辑模块划分，修改查找更方便

使用

  1. 将模块拆分后会有多个js文件，这些文件放在store/modules文件夹里

  2. 要使用相应模块时，就导入到核心文件里，配置module

  3. 如何确认
    可以在调试模块中看到除根模块外，还有子模块

访问子模块的state

尽管已经分模块了，但其实子模块的状态，还是会挂到根级别的 state 中，属性名就是模块名

访问方法：

  1. 通过模块名访问$store.state.模块名.模块中的state中的属性名
    因为子模块的状态还是挂在根级别的state中，所以要先进入state，再进入模块
    $store直接访问
  2. 通过mapState映射
    1. 默认根级别的映射 mapState([ ‘模块中的state中的属性名’ ])
    2. 子模块的映射 ：mapState(‘模块名’, [‘模块中的state中的属性名’]) - 需要开启命名空间 namespaced:true
      命名空间 namespaced:当不开启时，模块中的各个部分会合并到全局，可以直接全局访问，和根模块访问方法一样，但是这样容易冲突，建议开启
      开启命名空间

访问模块中的getters

使用模块中getters中的数据：

  1. 直接通过模块名访问$store.getters['模块名/getters中的方法名']
    含有/特殊字符可以通过使用[‘’]包裹的方式使用
  2. 通过mapGetters映射
    1.默认根级别映射mapGetters（[‘模块中的getters中的方法名’]）
    2.子模块的映射 ：mapGetters(‘模块名’, [‘模块中的getters中的方法名’]) - 需要开启命名空间 namespaced:true

访问模块中的mutations

  1. 注意：
    默认模块中的 mutation 和 actions 会被挂载到全局，需要开启命名空间，才会挂载到子模块。建议开启不然难以维护
  2. 调用方式：
    1. 直接通过 store 调用 $store.commit(‘模块名/mutation名 ‘, 额外参数)
    2. 通过 mapMutations 映射
      1. 默认根级别的映射 mapMutations([ ‘mutation名’ ])
      2. 子模块的映射 mapMutations(‘模块名’, [‘mutation名’]) - 需要开启命名空间

MCP、RAG、Skills、Agent、LLM（Skylark）五概念汇总

  1. LLM（大模型，例：Skylark云雀）

    • 是什么：AI的大脑，负责理解语言、逻辑推理、生成文字回答。
    • 作用：所有AI能力的基础，本身只有训练时学到的知识，不能直接操作本地文件、查私有资料。
    • 关系：Agent、RAG、Skills、MCP都要依赖大模型才能运行。

  2. MCP（模型上下文协议）

    • 是什么：一套统一的通信标准，AI用来连接本地资源的通用接口。
    • 作用：给AI装上手脚，让AI实时读取/修改本地文件、数据库、Git等硬件/本地资源；一次配置，多工具通用。
    • 特点：实时现场获取数据，需要配置文件授权访问范围。

  3. RAG（检索增强生成）

    • 是什么：知识库检索技术方案。
    • 作用：提前把文档、规范、手册切片向量化存入向量库，用户提问时自动检索相关资料，让AI依托真实资料回答，解决知识过时、AI幻觉编造问题。
    • 特点：用提前入库的静态资料做参考，不能直接修改本地文件。

  4. Skills（AI技能）

    • 是什么：标准化任务SOP（操作手册），一般用MD文档编写。
    • 作用：规定任务执行步骤、工具调用顺序、输出格式、业务规范，让AI固定流程干活，避免每次重复写提示词。
    • 特点：只约束“怎么做”，常搭配MCP、RAG使用。

  5. Agent（AI智能体）

    • 是什么：具备自主规划、决策、工具调用、反思纠错能力的智能执行主体。
    • 作用：总指挥，收到一个复杂目标后自动拆解步骤，自主判断调用MCP/RAG/Skills，循环执行直到任务完成，不需要人一步步指挥。
    • 定位：统筹前面所有能力的执行者。

  6. 形象类比串联

  • LLM（Skylark）= 人的大脑
  • MCP = 双手双脚，可以实时翻阅桌上的文件、操作电脑
  • RAG = 提前整理好的图书馆知识库，用来查阅权威资料
  • Skills = 岗位操作说明书，规定工作步骤和格式要求
  • Agent = 全能员工，自己看说明书、翻资料、动手操作，独立完成复杂工作
    Agent（员工）依托LLM（大脑），拿着Skills（操作手册），用MCP（手脚）操作本地资源，借助RAG（知识库）查询权威资料，自动完成复杂任务。

Skill

  1. Skills 是什么
    Skills = AI的可复用专业技能包 / 标准化工作手册
    本质：把固定任务流程、规则、输出格式、操作步骤、行业规范打包成一份结构化文档（核心是 SKILL.md ），AI可以重复调用这套工作方法稳定完成同类任务。
    通俗比喻：
  • MCP = 给AI装上手脚，让AI能摸到电脑文件、数据库、Git这些外部资源；
  • Skills = 给AI一本专业操作说明书，告诉AI拿到资源后，该按什么步骤、什么规范干活。

  1. Skills起到什么作用

  2. 告别重复写长提示词
    每次做代码评审、数据分析都要复制一堆要求，做成Skill后直接调用，不用重复写指令。
    2. 约束AI输出，避免胡说（防幻觉）
    规定好步骤、格式、校验规则，AI必须按流程执行，结果统一规范。
    3. 自动编排工具调用顺序
    搭配MCP使用时，Skill会告诉AI：先读取文件→再查数据库→最后整理报告，自动安排调用哪个MCP工具、调用顺序。
    4. 团队可共享复用
    一份代码规范Skill，团队所有人都能用，保证所有人的AI输出标准一致。

  3. 用在哪里

  4. 代码开发场景（你最常用）

    • 代码CR评审Skill：按前端规范检查Vue代码、给出优化清单
    • 项目配置校验Skill：自动读取 vue.config.js 、检查配置错误、输出修复方案
    • Git提交规范Skill：自动校验commit注释、生成提交说明

  5. 数据分析办公场景

    • Excel/CSV数据清洗、统计分析固定流程
    • 数据库查询规范：限定SQL写法、禁止删改语句

  6. 通用场景
    周报模板、竞品分析、接口文档自动生成等重复性工作。

  7. 怎么用

  8. 新建Skill文件夹
    全局Skill（所有项目通用）
    Windows： C:\用户\用户名.claude\skills\你的技能名\SKILL.md 
    Mac： ~/.claude/skills/你的技能名/SKILL.md 
    项目内Skill（仅当前项目生效）
    项目根目录： .claude/skills/code-review/SKILL.md 

  9. 编写SKILL.md（示例：前端代码评审技能）

  1. 使用方式
    重启软件后，直接用斜杠命令调用：
     /code-review 检查src下所有Vue文件 
    AI会自动加载这份Skill，按你写好的步骤调用MCP读取文件、完成代码评审。

  2. MCP VS Skills区别

    维度 MCP（模型上下文协议） Skills（技能包）
    定位 连接层：负责“能不能摸到外部资源” 流程层：负责“拿到资源后该怎么干活”
    本质 一套通用通信标准（JSON-RPC） 结构化的任务操作手册（Markdown文件）
    作用 提供文件、数据库、Git等工具能力 约束任务步骤、输出格式、业务规范
    关系 手脚 操作说明书，依赖MCP才能落地执行

一句话总结：
MCP让AI有能力拿数据，Skills教AI正确用好数据。

MCP

  1. MCP 是什么（Model Context Protocol，模型上下文协议）

  2. 它是一套通信协议、规则标准，就像 USB 协议、HTTP 协议，只是一套双方约定好的说话格式，看不见摸不着。

  3. 作用：规定 AI 客户端 和 本地工具之间怎么收发数据、怎么调用功能

  4. 简单说：规定了AI软件和电脑里的文件、数据库、Git、表格等外部资源之间，要用统一格式收发数据、怎么请求、怎么返回结果，大家都遵守同一套“说话规则”才能互相通信 。

  5. 底层基于  JSON-RPC 2.0  固定JSON报文格式通信，只要一方不遵守格式，双方就无法建立连接、无法交互，这就是协议强制统一的技术约束 。

  6. 三个核心角色（必须分清）

1. Host（宿主）：你用的AI软件，比如 Cursor、Claude桌面端、VSCode，是发起任务的一方。
2. MCP Client（客户端）：AI软件内置的连接器，负责把自然语言翻译成标准MCP请求报文。
3. MCP Server（服务端）：一个个具体能力工具（读取本地文件、操作数据库、读取Git日志），遵守MCP协议对外提供功能。

你写的 mcp.json 只是启动配置清单：告诉AI要启动哪些MCP服务、授权访问哪个文件夹，不属于协议本身，只是使用MCP的配置文件而已。

  1. MCP 起到什么作用

1. 打破AI孤岛，让AI拥有操作本地资源的能力
没MCP前，AI只能接收你复制粘贴的文字；有了MCP，AI可以直接读取、修改项目代码、查询数据库、解析PDF，不用手动传文件。
2. 一次开发，全生态通用，降低重复开发成本
一套文件读取MCP服务，Claude、Cursor所有支持MCP的AI都能直接用，不用为每个软件单独写对接代码。
3. 权限可控，保障隐私安全
可以限定AI只能访问你指定的文件夹、数据表，不能随意读取电脑全盘文件。
4. 标准化工具调用，方便扩展各类能力
社区有上千款现成MCP工具，配置一下就能给AI新增能力：查日志、操作表格、连接GitHub等。

  1. MCP 用在哪里

    1. 常用软件
      Cursor编辑器、Claude桌面端、VSCode（Cline插件）、Windsurf等主流AI开发工具。
    2. 高频使用场景
      1. 前端开发：AI自动遍历Vue项目、检查配置错误、代码评审、Git提交管理、批量重构代码。
      2. 数据处理：连接本地数据库执行SQL、分析Excel/CSV表格、解析日志文件做统计。
      3. 办公场景：批量总结PDF、整理文件夹、同步网盘文档、自动生成周报文档。

  2. MCP 怎么用

1. 安装支持MCP的软件（推荐Cursor或Claude），同时装好Node.js环境。
2. 在软件指定目录新建 mcp.json 配置文件，填写要启动的MCP服务、授权访问的文件夹路径。
3. 重启软件，AI会自动拉起MCP服务，基于统一JSON-RPC协议建立通信。
4. 直接用自然语言下发指令，AI自动调用对应的MCP工具完成任务。

  • 举个完整流程例子
    你在Cursor输入：帮我检查项目里 vue.config.js 的配置错误

    1. Cursor（Host）通过内置MCP客户端，发送标准JSON格式请求：读取D盘项目下的vue.config.js文件；
    2. 文件MCP服务端收到标准指令，读取本地文件内容；
    3. 再按照MCP规定的JSON格式，把文件内容返回给AI；
    4. AI拿到代码内容分析，输出错误和优化方案。

  1. 补充：MCP为什么能让所有人遵守规范？

没有行政强制约束，靠两点天然约束：

1. 技术兼容约束：只有双方报文格式完全符合MCP规范，才能握手建立连接；格式写错直接通信失败、连接断开，工具无法使用。
2. 生态利益约束：主流AI工具全部支持MCP，开发、使用MCP工具可以一次适配全平台，不用重复造轮子，开发者自然主动遵守官方标准，同时官方提供各类编程语言SDK，默认就遵循协议规范。

RAG和Skills对比

  1. 区别
    RAG：给AI提供「资料、事实数据」（解决：要参考什么内容）
    Skills：给AI提供「流程、操作方法、规范步骤」（解决：这件事该怎么做）

  2. 什么是 RAG

RAG = 检索增强生成

  • 本质：私有知识库查询方案
  • 原理：提前把大量PDF、文档、手册、接口文档向量化存入向量库；你提问时，AI自动检索匹配最相关的资料片段，拿着真实资料再回答，防止AI编造、解决知识过时问题。
  • 核心回答：What（是什么、有哪些数据、规则内容是什么）
  • 例子：查公司接口文档、前端开发规范原文、项目历史需求文档。

  1. 什么是 Skills（AI技能）

    • 本质：可复用的标准化任务操作手册（SOP），一般是 SKILL.md 结构化文档
    • 原理：规定任务执行步骤、工具调用顺序、输出格式、校验规则，告诉AI先做什么、后做什么、用哪个MCP工具、结果要怎么排版展示。
    • 核心回答：How（怎么做、按什么流程执行）
    • 例子：Vue代码评审流程、Git规范提交步骤、数据库查询的操作规范。

  2. 详细对比表

    对比维度 RAG Skills
    核心作用 补充事实知识、参考资料 约束任务流程、执行方法、输出格式
    存储内容 海量业务文档、手册、原始数据 步骤、规范、SOP、工具调用顺序
    工作方式 相似度模糊检索，自动匹配相关文档 按任务名称精准调用技能，流程固定
    依赖组件 需要向量数据库、文档切片、向量化模型 不需要数据库，纯结构化指令文件，可搭配MCP使用
    典型场景 查技术文档、法律条文、历史业务资料问答 代码自动化检查、批量处理、固定流程办公任务
    能不能操作文件 只能读取提前入库的文档，不能实时修改本地文件 搭配MCP可以实时读写、修改本地代码、操作数据库

  3. 同场景对比

场景：让AI检查Vue项目代码规范

1. RAG负责：
检索《前端ESLint编码规范》《Vue3开发手册》原文，把规范条文给到AI，告诉AI正确的规则是什么。
2. Skills负责：
规定执行步骤：
①用MCP读取所有 .vue 文件
②逐条对照规范检查
③用表格输出「问题位置+错误原因+修改方案」
也就是告诉AI按什么顺序、用什么工具、以什么格式完成检查任务。

  1. 三者联动（MCP+Skills+RAG）

1. MCP：AI的手脚，实时读取本地项目文件
2. RAG：AI的参考书库，调取官方编码规范资料
3. Skills：AI的工作说明书，规定先查资料→再读代码→最后格式化输出结果

  1. 使用选择建议

1. 你有几百份、上千份文档需要随时查询、查原文资料 → 用 RAG
2. 你需要重复做固定流程的任务、规范AI操作步骤、统一输出格式 → 用 Skills
3. 想要又查资料、又自动干活 → RAG + Skills + MCP 组合使用

Agent

  1. Agent 是什么（AI智能体）

简单说：具备自主规划、思考、决策、调用工具能力的AI智能助手。
普通大模型只会根据你当前一句话直接回答；
Agent 可以自己拆解任务、自己判断需要调用什么工具、一步步自主完成复杂任务，不需要你反复引导、分步下达指令。

核心特点

1. 能思考拆解任务：把一个大需求拆成多个小步骤
2. 能自主选择工具：判断该用MCP、RAG还是其他工具
3. 能循环执行+纠错：结果不对就重新调用工具、再次分析，直到任务完成
4. 可以结合Skills设定执行规范，约束自己的工作流程

  1. 举例

需求：做一份项目代码安全检测报告

  • 普通AI：只会让你把代码一段段粘贴过来，你一步步指挥才会做
  • Agent：
    1. 自己规划步骤：先读取项目配置文件 → 扫描所有Vue、JS代码 → 检索安全编码规范（RAG）→ 整理问题 → 生成表格报告
    2. 自主调用MCP读取本地文件，调用RAG查安全规范
    3. 发现缺文件就自动查找目录，出错会重试，全程不用你一步步指挥
  1. Agent 组件构成

1. 大模型（大脑，如Skylark）：负责思考、任务拆解、决策
2. 工具集（MCP、API、数据库等）：Agent用来获取外部信息、操作资源的手脚
3. 记忆模块：记住历史对话、已执行过的步骤，避免重复操作
4. 规划模块：拆解复杂任务，制定执行顺序
5. 反思校验：执行完检查结果是否满足需求，不满足就重试优化

  1. 和前面几个概念的关系

1. MCP：给Agent提供可以操作本地文件、数据库的工具能力（手脚）
2. RAG：给Agent提供参考知识库，保证决策有依据、不幻觉（参考书）
3. Skills：给Agent规定固定工作流程、输出格式、业务规则（工作手册）
4. Agent：统筹以上所有组件的总指挥，自主安排先调用谁、后调用谁，自动完成整套复杂工作

  1. 常见使用场景

1. 开发类：自动搭建项目、批量重构代码、自动化测试、Git批量操作
2. 办公类：读取多份文档汇总分析、自动整理周报、数据报表生成
3. 运维类：读取日志排查错误、批量检查服务器配置

  1. 总结

普通AI：你说一步，做一步；
Agent：你只说最终目标，它自己规划步骤、调用各类工具，自主把整件事做完。
